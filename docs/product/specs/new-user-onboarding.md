# 신규 사용자 온보딩

## 문제

처음 방문한 사용자는 Matchuri가 무엇을 해주는지 빠르게 이해하고, 가능한 한 짧은 시간 안에 첫 번째 유의미한 결과에 도달해야 합니다.

## 사용자

아직 취향이나 선호 정보를 설정하지 않은 첫 방문 사용자

현재 기준에서는 아래 사용자도 포함합니다.

- 로그인은 했지만 필수 약관 동의를 완료하지 않은 신규 사용자

## 목표

혼란을 최소화하면서 사용자가 첫 번째 핵심 행동을 완료하도록 돕습니다.

현재 기준에서 첫 번째 핵심 행동은 아래를 포함합니다.

- 로그인 또는 가입 완료
- 필수 약관 2종 동의 완료
- 닉네임 설정 완료
- 이후 첫 추천 흐름 또는 다음 온보딩 단계 진입

## 비목표

- 완전한 개인화 제공
- 고급 추천 세부 조정 기능 제공

## 사용자 흐름

1. 사용자가 제품에 처음 진입합니다.
2. 사용자는 제품의 핵심 가치를 이해합니다.
3. 사용자는 로그인 또는 회원 가입을 완료합니다.
4. 사용자는 필수 약관 2종 `서비스 이용약관`, `개인정보 처리방침`에 동의합니다.
5. 사용자는 첫 결과를 만드는 데 필요한 최소 정보만 입력합니다.
6. 사용자는 신뢰할 수 있거나 추가로 조정할 수 있는 첫 결과를 확인합니다.

## 현재 확정 기준

- 필수 약관 동의는 온보딩 경로 안에 포함됩니다.
- 자체 회원가입 기본 경로는 계정 생성과 필수 약관 동의, 닉네임 입력을 하나의 요청으로 처리합니다.
- 소셜 로그인 신규 가입과 레거시 자체 회원가입 흐름에서는 계정 생성 이후 약관 동의와 닉네임 설정 단계를 따릅니다.
- 미동의 회원도 로그인은 가능하지만, 핵심 서비스 기능 진입 전에는 약관 동의 또는 닉네임 설정 화면으로 이동합니다.
- 서버가 생성한 소셜 회원 임시 닉네임은 온보딩 완료 전 표시용 placeholder이며, 사용자가 닉네임을 확정해야 핵심 서비스 기능에 진입할 수 있습니다.
- 프론트는 로그인, refresh, OAuth2 교환, 약관 동의 제출, 닉네임 변경 응답의 `data.onboarding.nextStep`을 기준으로 다음 온보딩 화면을 결정합니다.

## UX 원칙

- 로그인 성공 후 다음 행동이 명확해야 합니다.
- 자체 회원가입에서는 필수 약관 동의와 닉네임 입력이 가입 완료 전에 끝나야 합니다.
- 소셜 로그인과 레거시 자체 로그인에서는 필수 약관 동의가 우회 가능한 숨은 단계가 아니라, 사용자가 지금 완료해야 할 작업으로 분명히 보여야 합니다.
- 필수 약관 동의와 닉네임 설정을 마치기 전에는 취향 입력, 추천, 그룹 기능 진입으로 흐름이 분산되지 않아야 합니다.
- 약관 동의 화면은 현재 필요한 2개 항목만 보여주고 과도한 입력 부담을 만들지 않습니다.

## 완료 기준

- 첫 화면에서 제품 목적이 쉬운 말로 설명됩니다.
- 온보딩 경로는 정말 필요한 정보만 묻습니다.
- 자체 회원가입에서는 가입 요청 안에서 필수 약관 동의와 닉네임 입력이 함께 끝납니다.
- 소셜 로그인과 레거시 자체 로그인에서는 로그인 후 필수 약관 동의와 닉네임 설정 화면으로 자연스럽게 이어집니다.
- 필수 약관 동의와 닉네임 설정을 완료하기 전에는 핵심 서비스 화면으로 잘못 진입하지 않습니다.
- 사용자는 문서를 읽지 않아도 첫 결과에 도달할 수 있습니다.
