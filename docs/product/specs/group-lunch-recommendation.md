# 그룹 점심 메뉴 결정

## 문제

여러 명이 함께 점심을 먹을 때 메뉴를 정하는 과정은 길고 피로합니다.
각자의 취향, 비선호, 제한 재료가 다르기 때문에 단순 검색이나 인기순 나열만으로는 빠르게 합의하기 어렵습니다.
결국 논의가 길어지거나, 일부 취향이 무시되거나, 늘 익숙한 메뉴로 타협하게 됩니다.

## 사용자

- 함께 점심 메뉴를 정해야 하는 직장인 팀
- 여러 명이 동시에 의견을 조율해야 하는 학생 그룹
- 메뉴를 빨리 정하고 싶지만 취향 반영도 포기하고 싶지 않은 사용자

## 목표

그룹 구성원이 각자의 취향을 큰 부담 없이 반영하고, 후보 메뉴를 빠르게 확인한 뒤, 투표를 통해 납득 가능한 최종 메뉴를 결정하도록 돕습니다.

## 비목표

- 완벽한 AI 기반 최적 메뉴 도출
- 음식점 리뷰 플랫폼 수준의 장소 탐색 기능
- 고급 협업 필터링 추천
- 모든 상황을 커버하는 범용 회식/모임 의사결정 도구

## 핵심 시나리오

1. 사용자가 그룹을 생성합니다.
2. 다른 사용자가 초대 링크 또는 코드로 그룹에 참여합니다.
3. 각 사용자는 저장된 취향을 불러오거나 필요한 취향을 입력합니다.
4. 방장이 그룹 추천 준비 세션을 시작합니다.
5. 각 그룹원은 취향을 확인/수정한 뒤 준비 완료를 누릅니다.
6. 모든 현재 활성 그룹원이 준비 완료하면 시스템은 그룹 취향을 바탕으로 후보 메뉴 3개 안팎을 제시합니다.
7. 그룹 구성원은 후보 메뉴에 투표합니다.
8. 시스템은 투표 결과를 바탕으로 최종 메뉴를 확정합니다.
9. 필요하면 확정된 메뉴를 기반으로 주변 가게를 확인합니다.

## 범위

- 그룹 생성
- 그룹 목록/상세 조회
- 초대 링크 또는 코드 기반 그룹 참여
- 구성원별 취향 입력 또는 저장 취향 불러오기
- 그룹 추천 생성
- 후보 메뉴 목록 제공
- 그룹 투표
- 최종 메뉴 확정

## 범위 제외

- 장소 추천 고도화
- 실시간 외부 배달/주문 연동
- 투표 없이 자동 최종 결정되는 고급 추천
- 추천 성능 분석 대시보드

## 사용자 흐름 요약

### 진입

- 사용자는 그룹을 새로 만들거나 기존 그룹에 참여합니다.

### 준비

- 참여자는 본인 취향을 확인합니다.
- 저장된 취향이 없으면 간단한 취향 입력을 먼저 진행합니다.
- 방장이 추천 준비 세션을 열면 참여자는 준비 완료 전까지 취향을 수정할 수 있습니다.
- 모든 현재 활성 참여자가 준비 완료하면 후보 추천 단계로 자동 전환됩니다.

### 추천

- 그룹 추천이 시작되면 후보 메뉴가 제시됩니다.
- 각 후보에는 최소한의 설명 또는 선택 힌트가 함께 제공됩니다.

### 합의

- 사용자는 후보에 투표합니다.
- 누가 참여 중인지와 투표 진행 상태가 명확히 표시됩니다.
- 시스템은 투표 결과로 최종 메뉴를 확정합니다.

## 완료 기준

- 사용자는 그룹을 생성하거나 참여할 수 있습니다.
- 저장된 취향이 없을 때도 그룹 추천 흐름을 막지 않습니다.
- 시스템은 그룹 기준 후보 메뉴를 3개 안팎으로 제시합니다.
- 각 참여자는 후보 메뉴에 투표할 수 있습니다.
- 최종 메뉴가 하나로 확정되고 화면에서 명확히 표시됩니다.
- 사용자는 현재 그룹 방 상태와 그룹 추천 진행 상태를 혼동하지 않습니다.

## 성공 지표 초안

- 그룹 메뉴 결정 완료까지 걸리는 시간
- 그룹 추천 시작 대비 최종 확정 완료 비율
- 추천 후보 확인 후 실제 투표 참여율
- 최종 결과에 대한 사용자 주관 만족도

## 가정

- MVP에서는 그룹 추천의 핵심 가치가 "정교한 추천 정확도"보다 "빠른 합의 도출"에 있습니다.
- MVP에서는 후보 수를 제한하는 것이 사용성에 유리합니다.
- 장소 정보는 핵심 기능이 아니라 최종 메뉴 이후 보조 기능으로 다룹니다.
