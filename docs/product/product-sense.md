# 제품 판단 기준

이 문서는 구현 중 제품 판단이 필요할 때 보는 최소 기준입니다.
사람이 읽는 프로젝트 소개와 제품 서사는 GitHub Wiki에서 관리합니다.

## 한 줄 정의

Matchuri는 개인과 그룹이 점심 메뉴를 더 빨리, 덜 스트레스받고, 더 납득 가능하게 결정하도록 돕는 서비스입니다.

## 핵심 전제

- Matchuri는 맛집 검색 서비스가 아니라 메뉴 결정 비용을 줄이는 서비스입니다.
- MVP의 핵심 흐름은 `후보 3개 안팎 + 투표 + 최종 메뉴 확정`입니다.
- 추천 정확도보다 사용자가 결정을 끝내는 흐름이 우선입니다.
- 장소 정보는 메뉴 결정 이후의 보조 레이어로 다룹니다.

## 우선순위

우선 구현:

- 회원 가입과 로그인
- 취향 저장과 불러오기
- 개인 추천 기본 흐름
- 그룹 생성, 초대, 참여
- 그룹 추천 후보 제시
- 투표와 최종 메뉴 확정

후순위:

- 협업 필터링 기반 고급 추천
- 추천 정확도 분석 대시보드
- 고도화된 맛집 정렬과 지도 최적화
- 모든 모임 상황을 커버하는 범용 의사결정 도구

## 판단 원칙

- 입력 부담을 늘려 추천을 정교하게 만드는 것보다 첫 결과까지 가는 단계를 줄입니다.
- 후보는 너무 많지 않게 유지합니다.
- 그룹 기능에서는 복잡한 알고리즘보다 모두가 이해하는 상태 흐름을 우선합니다.
- 결과에는 사용자가 납득할 수 있는 추천 이유 힌트가 있어야 합니다.
- 구현 선택이 애매하면 "점심 메뉴 결정 비용을 줄이는가?"로 판단합니다.
