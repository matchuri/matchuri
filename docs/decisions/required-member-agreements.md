# 필수 약관 동의 구조 결정

이 문서는 필수 약관 동의 구현 기준만 남깁니다.
사람이 읽는 계정/온보딩 설명은 GitHub Wiki에서 관리합니다.

## 결정

- 필수 약관 동의는 별도 `member_agreements` 테이블로 관리합니다.
- `members`에는 약관 동의 여부를 직접 누적하지 않습니다.
- 서비스 이용 가능 여부는 "회원이 현재 필수 약관 2종의 최신 필수 버전에 모두 동의했는가"로 판단합니다.
- 미동의 회원도 로그인은 가능하지만, 핵심 API는 차단합니다.
- 프론트는 로그인 직후 약관 동의 화면으로 이동합니다.

## 필수 약관

현재 필수 약관은 두 종류입니다.

- `TERMS_OF_SERVICE`
- `PRIVACY_POLICY`

최신 필수 버전은 초기 MVP에서는 서버 상수로 관리합니다.

## 데이터 기준

권장 테이블:

- `member_agreements`

핵심 컬럼:

- `member_id`
- `agreement_type`
- `agreement_version`
- `agreed_at`

권장 제약:

- `(member_id, agreement_type, agreement_version)` unique

## 접근 제어 기준

허용:

- 로그인
- 로그아웃
- 약관 상태 조회
- 약관 동의 제출

필수 약관 완료 후 허용:

- 취향 프로필 관리
- 개인 추천
- 그룹 생성/참여
- 그룹 추천
- 투표

## 현재 하지 않는 것

- 선택 약관 관리
- 마케팅 수신 동의
- 약관 전문 DB/CMS 관리
- 관리자 약관 버전 관리 화면
- 철회/재동의 이력의 복잡한 상태 머신
