# 헬스 체크 API

이 문서는 Matchuri 백엔드의 공개 헬스 체크 엔드포인트 계약을 설명합니다.
목적은 배포 후 기본 가용 상태를 빠르게 확인하고, 프론트나 운영자가 서버 생존 여부를 같은 기준으로 해석하도록 만드는 것입니다.

## 범위

- 백엔드 프로세스의 기본 응답 가능 여부
- 공개 엔드포인트 기준의 최소 상태 확인

## 비범위

- 상세 내부 진단 정보
- 데이터베이스, 외부 연동, 스레드풀 등의 운영 상세 지표
- 장애 원인 분석용 내부 상태 노출

## 엔드포인트

- Method: `GET`
- URL: `/api/v1/health`
- 권한: 비회원
- 설명: 백엔드 서비스의 기본 가용 상태를 확인합니다.

## 응답 예시

```json
{
  "groups": [
    "liveness",
    "readiness"
  ],
  "status": "UP"
}
```

## 응답 해석 기준

- `status`
  - `UP`: 기본 서비스 상태가 정상입니다.
  - `DOWN`: 기본 서비스 상태가 비정상입니다.
- `groups`
  - Actuator health group 정보입니다.
  - 현재는 `liveness`, `readiness`가 노출될 수 있습니다.

## 공개 정책

- 이 엔드포인트는 공개 API이므로 상세 내부 상태는 응답에 포함하지 않습니다.
- 민감한 설정값, 예외 메시지, 인프라 세부 정보는 노출하지 않습니다.
- 운영 상세 진단은 별도 내부 도구 또는 로그 기준으로 확인합니다.

## 사용 예시

- 로컬 확인:

```text
GET http://localhost:8080/api/v1/health
```

- 배포 점검:
  - 애플리케이션 기동 직후 `status=UP`인지 확인합니다.
  - 외부 헬스체크 대상 엔드포인트로 사용할 수 있습니다.
